# carolinemeijerinkTPSI0226
Final Project

Tema escolhido: Sistema de reservas de hotel

O projeto desenvolvido consiste num sistema completo de gestão de reservas para o Hotel XPTO, concebido integralmente na linguagem de programação Python com uma interface em linha de comandos (CLI). Esta aplicação foi desenhada com o propósito de automatizar e otimizar os processos operacionais de um estabelecimento hoteleiro, permitindo o controlo de hóspedes e estadias. Através de uma arquitetura modular, o sistema separa de forma clara e bem organizada as responsabilidades associadas à interface com o utilizador, à validação de dados e ao armazenamento da informação, assegurando a integridade e a escalabilidade do software.

No cerne do funcionamento da aplicação encontra-se a entidade responsável por modelar cada reserva, a qual armazena os dados cruciais do cliente, como o nome completo, o NIF, o e-mail e os períodos de check-in e check-out. Cada registo efetuado recebe de forma estritamente automática um identificador único numérico. Este mecanismo foi desenhado para evitar duplicações e conflitos de dados, ajustando-se mesmo após o encerramento e reinicialização da aplicação através de uma leitura dinâmica do maior ID previamente guardado. Sempre que um utilizador inicia o processo de criação de uma reserva, o sistema exige o cumprimento de parâmetros de integridade através de expressões regulares (Regex). Esta camada de validação certifica que o NIF contém precisamente nove dígitos numéricos, que o endereço de e-mail respeita os padrões internacionais de sintaxe e que as datas de permanência são válidas no calendário gregoriano, garantindo ainda, por via de lógica temporal, que o check-out ocorre sempre em momento posterior ao check-in.

Para além da recolha e validação estruturada de informação, o sistema disponibiliza um conjunto de ferramentas para a manipulação e pesquisa de dados. Os utilizadores podem listar a totalidade dos registos ativos ou realizar buscas direcionadas através de dois algoritmos de pesquisa distintos. O primeiro é uma pesquisa linear aplicada ao nome do hóspede, ideal para localizações parciais ou buscas textuais flexíveis. O segundo consiste numa rotina de pesquisa binária concebida para encontrar registos de forma altamente eficiente através do identificador único (ID). Dado que a pesquisa binária exige que a coleção de dados esteja perfeitamente ordenada, a aplicação integra uma rotina manual baseada no algoritmo Bubble Sort, assegurando o correto ordenamento numérico antes da execução da busca. O utilizador pode também invocar esta ordenação para organizar a visualização das suas reservas no menu principal.

A manutenção dos registos é assegurada através de operações completas de atualização e remoção. O módulo de edição permite alterar os atributos de uma reserva já existente, reavaliando os novos dados introduzidos sob as mesmas regras de validação inicial. Complementarmente, a remoção de registos foi desenhada com uma camada de segurança que exige uma confirmação explícita por parte do operador antes de eliminar em definitivo qualquer informação do sistema, prevenindo a perda acidental de dados de hóspedes. Adicionalmente, o programa conta com um painel de processamento estatístico capaz de calcular indicadores de volume global e fornecer métricas quantitativas sobre a ocupação do hotel.

Por fim, a persistência de dados é uma das principais funcionalidades deste projeto, sendo gerida através da serialização de objetos no formato de ficheiro JSON. Toda e qualquer alteração efetuada na base de dados, seja a criação, modificação ou eliminação de uma reserva, é imediatamente refletida no ficheiro físico de armazenamento. Ao arrancar, a aplicação faz a leitura inversa, deserializando o ficheiro e reconstruindo com precisão o estado anterior do sistema. Esta abordagem assegura que o fluxo de trabalho do Hotel XPTO nunca é interrompido e que toda a informação histórica é preservada de forma perfeitamente segura entre sessões de utilização.
